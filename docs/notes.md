Git is a Version Control System where we can work on our project in a structured way. Our progress, and project can be tracked fully. It provides the features like staging, commit, push etc. such that we can easily work on and maintain our projects with history. Version Controlling is important in Software Development so that we can track, roll-back and store our project files and folders in an organized way. It helps to do collaborate with the team developers and build applications in an easier way.

git clone: it is used to download the git repo from cloud to local device along with version history.
git status: it is used to view the current state of the repo like staging, commit, etc.
git add: it is used to move the file from development to the staging state.
git commit: once the projet is staged, it is committed using this command very we put short msg of what had done.
git branch: it is used to view all the branches and see which branch we are currently in.
git checkout: it is used to switch the branch.
git merge: it is used to merge another branch into the current branch.
git pull: it is used to fetch plus merge the cloud repo into the local.
git push: it is used to push the local project into the cloud.
